## Typography

Inherits the Inspector drawer's four-size scale. No new sizes introduced.

| Role | Size | Tailwind | Weight | Line-Height | Usage in Phase 9 |
|------|------|----------|--------|-------------|------------------|
| Section heading | 14px | `text-sm` | 600 | 1.25 | "Relationships" tab heading; graph-page h1 "Relationship Graph"; H5 modal h2 |
| Body / label | 12px | `text-xs` | 400 | 1.5 | Counterparty DID (mono); numeric valence/weight at H2; warmth-bucket label at H1; empty-state copy |
| Meta / caption | 11px | `text-[11px]` | 400 | 1.4 | Recency tick caption ("Last interaction: tick N"); graph legend; H1 bucket note "Numeric weights require H2 elevation" |
| Monospace DID / hash | 12px | `font-mono text-xs` | 400 | 1.3 | Counterparty DID in relationship row; edge hash in H5 modal; raw audit payload preview |

**Weight discipline:** Only 400 (regular) and 600 (semibold). No 500 (medium), no 700 (bold). Matches Phase 6–8 precedent.

**Monospace usage:** DIDs, hashes, tick numbers, raw payload snippets render in JetBrains Mono — never in Inter. Numeric warmth values at H2 render in Inter (`text-xs`) to mark them as semantic content, not identifiers.

**Line-height discipline:** `leading-relaxed` (1.625) is reserved for multi-sentence warnings (inherited Phase 8). Phase 9 adds no warning paragraphs — all body copy uses the default `leading-normal` (1.5).

### Warmth scale (NEW — Phase 9, scoped to relationship surfaces)

Phase 9 binds three semantic warmth colors. These reuse existing Tailwind palette steps — NO new globals.css tokens.

| Bucket | Tailwind Token | Hex | Usage |
|--------|---------------|-----|-------|
| `cold` | `text-neutral-400` / `bg-neutral-400` / `stroke-[#9ca3af]` | #9CA3AF (neutral-400; SVG stroke uses same) | Inspector dot + H1 bucket label + graph edge stroke for `weight < 0.20` |
| `warm` | `text-amber-400` / `bg-amber-400` / `stroke-[#f59e0b]` | #F59E0B (amber-500 hex; token `text-amber-400` resolves to #FBBF24 — see reconciliation below) | Inspector dot + H1 bucket label + graph edge stroke for `0.20 ≤ weight < 0.60` |
| `hot` | `text-rose-400` / `bg-rose-400` / `stroke-[#e11d48]` | #FB7185 (rose-400 token); SVG stroke uses #E11D48 (rose-600) for edge contrast on dark bg | Inspector dot + H1 bucket label + graph edge stroke for `weight ≥ 0.60` |

**Reconciliation with RESEARCH.md §Graph Layout Algorithm:** RESEARCH.md stroke colors (`cold=#9ca3af`, `warm=#f59e0b`, `hot=#e11d48`) are the authoritative SVG stroke hex values for the graph page. These map to the Tailwind tokens `neutral-400`, `amber-500`, `rose-600` (chosen for edge visibility on `#0A0A0A` bg, not the text-token equivalents). Inspector inline dots and text labels use the Tailwind tokens `text-neutral-400`, `text-amber-400`, `text-rose-400` (slightly brighter for small-dot visibility). Two-surface two-hex pattern is intentional and mirrors Phase 8's `border-rose-900` vs `text-red-500` distinction.

### Accent reserved for (explicit list — never "all interactive elements")

Warmth accent (`amber`/`rose`) is reserved for:
1. Relationship-row dot indicator (Inspector Relationships tab)
2. Relationship-row bucket label at H1 (Inspector Relationships tab)
3. Graph page SVG edge stroke (`/grid/relationships`)
4. Graph page SVG node label halo when hovering (dim variant: `fill-opacity="0.15"`)
5. Graph page legend (3 swatches + bucket name only)

Warmth accent MUST NOT appear on:
- Buttons (use existing brand `--color-accent` for interactive affordances)
- Inspector drawer border-left (stays `border-neutral-800`)
- Tab-bar active underline (stays `border-accent` #7DD3FC)
- The H2 elevation button (uses inherited Phase 6 blue H2 tier color `#60A5FA`)
- Any copy other than bucket labels and legend

### Tier chip colors (inherited Phase 6, unchanged)

H2 elevation affordance inside the Inspector Relationships tab uses the v2.1-frozen tier chip palette:

| Tier | Tailwind Token | Hex | Usage |
|------|---------------|-----|-------|
| H1 | `text-neutral-400` | #A3A3A3 | Default tier chip (no warmth accent elevation) |
| H2 | `border-blue-400 text-blue-400` | #60A5FA | "Reveal numeric weights (H2)" button |

### H5 per-edge inspector — destructive framing NOT applied

H5 per-edge inspection in Phase 9 is a **read** action (not destructive — no audit-chain mutation beyond the `operator.inspected` entry already emitted for H2 memory queries). Per 09-CONTEXT.md §specifics REL-04: "Not required — read is non-destructive; H5 gate suffices without copy-verbatim lock." The H5 edge-events modal therefore does NOT reuse IrreversibilityDialog's `border-rose-900` destructive frame. It uses a plain `border-neutral-800` frame. Tier chip color remains H5 `border-rose-400 text-rose-400` as inherited — same as Phase 8 — on the tier indicator badge only, not the modal frame.

### WCAG 2.1 AA compliance

| Pair | Contrast | Verdict |
|------|----------|---------|
| `text-neutral-400` #A3A3A3 on `bg-neutral-950` #0A0A0A | ~7.2:1 | AA pass |
| `text-neutral-600` #525252 on `bg-neutral-950` #0A0A0A | ~3.8:1 | AA pass for `text-[11px]` decorative captions ONLY; NOT for instructional copy |
| `text-amber-400` #FBBF24 on `bg-neutral-950` #0A0A0A | ~11.0:1 | AA pass |
| `text-rose-400` #FB7185 on `bg-neutral-950` #0A0A0A | ~6.3:1 | AA pass |
| `text-blue-400` #60A5FA on `bg-neutral-950` #0A0A0A | ~6.4:1 | AA pass |
| SVG stroke `#F59E0B` on `#0A0A0A` | ~9.8:1 | AA pass for 1.5px line (treated as graphical object per WCAG 1.4.11) |
| SVG stroke `#E11D48` on `#0A0A0A` | ~4.9:1 | AA pass |
| SVG stroke `#9CA3AF` on `#0A0A0A` | ~7.2:1 | AA pass |

**Color is never the sole indicator:** each warmth bucket also carries a text label ("cold", "warm", "hot") — accessibility per WCAG 1.4.1. The graph page includes a legend mapping each swatch to its label. Hover/focus states on edges add `stroke-dasharray="2,2"` as a non-color differentiator.

### Primary CTA labels

| Surface | CTA | Label | Notes |
|---------|-----|-------|-------|

### Destructive confirmations

**None.** Phase 9 is a derived-view phase. No delete, no mutate, no force-action. H2 elevation reuses existing ElevationDialog copy (verbatim-locked Phase 6). H5 edge-events read does NOT require IrreversibilityDialog — per 09-CONTEXT.md §specifics REL-04 read is non-destructive. Zero-allowlist-growth invariant (D-9-13) means no new audit events at all; the H2/H5 endpoints emit the existing `operator.inspected` (Phase 6).

### Copywriting style guide (inherited Phase 8)

- No exclamation marks. No emoji. No "Oops" or "Uh oh."
- Active voice: "This Nous has not yet spoken…" not "Speaking has not occurred…"
- Named nouns: "Nous" (capital N), "Grid" (capital G), "tick" (lowercase), "Relationships" (capital R as proper noun for the feature surface).
- Numeric precision: warmth numerics at H2 always render `toFixed(3)` with sign for valence (`+0.420` / `-0.130`) and no sign for weight (`0.780`).
- Avoid temporal vague words: say "tick N" not "recently". The recency caption says "Last interaction: tick {N}" because replay rigs operate on ticks, not wall-clock.

### Surface 1b: Inspector Tab-Bar — add "Relationships" tab

**File (EDIT):** `dashboard/src/app/grid/components/tab-bar.tsx` OR integrate into existing Inspector tab enumeration (codebase scan shows `tab-bar.tsx` is the region-switcher, not the Inspector section switcher).

**Finding during research:** Existing Inspector (`inspector.tsx`) stacks all sections vertically (Psyche, Thymos, Telos, Memory) — no tab switcher. Phase 9 introduces a scoped tab surface **only within the Inspector drawer** to segregate Relationships from the existing stacked sections without overwhelming the drawer.

**Decision (auto-default — conservative):** Add a new lightweight tab strip at the top of the Inspector drawer body, BELOW the heading, ABOVE the stacked sections:

```
┌─────────────────────────────────────────┐
│ Nous Inspector               [close X]  │  existing header (unchanged)
│                                         │
│ ┌─────────┐ ┌──────────────┐           │  NEW tab strip — text-xs, h-9
│ │ Overview│ │ Relationships│           │  active: border-b-2 border-accent text-neutral-100
│ └─────────┘ └──────────────┘           │  inactive: border-b border-neutral-800 text-neutral-400
│                                         │
│ [existing stacked sections OR           │
│  Relationships section — switched]      │
└─────────────────────────────────────────┘
```

**Tab strip classes:**

```tsx
<div
  role="tablist"
  data-testid="inspector-tabs"
  className="flex gap-0 border-b border-neutral-800"
>
  <button
    role="tab"
    data-testid="inspector-tab-overview"
    aria-selected={activeTab === 'overview'}
    onClick={() => setActiveTab('overview')}
    className={`px-3 py-2 text-xs transition-colors focus:outline-none focus:ring-2 focus:ring-neutral-500 ${
      activeTab === 'overview'
        ? 'border-b-2 border-accent text-neutral-100 -mb-[1px]'
        : 'text-neutral-400 hover:text-neutral-200'
    }`}
  >
    Overview
  </button>
  <button
    role="tab"
    data-testid="inspector-tab-relationships"
    aria-selected={activeTab === 'relationships'}
    onClick={() => setActiveTab('relationships')}
    className={`px-3 py-2 text-xs transition-colors focus:outline-none focus:ring-2 focus:ring-neutral-500 ${
      activeTab === 'relationships'
        ? 'border-b-2 border-accent text-neutral-100 -mb-[1px]'
        : 'text-neutral-400 hover:text-neutral-200'
    }`}
  >
    Relationships
  </button>
</div>
```

**Interaction:**
- Default tab: `overview` (current stacked-sections view — zero visual change for users not caring about relationships).
- Tab state is LOCAL to the Inspector instance (`useState<'overview' | 'relationships'>`), NOT persisted — resets to `overview` each time a new DID is selected.
- Keyboard: Left/Right arrow keys cycle tabs (WAI-ARIA tablist pattern). Tab/Shift+Tab moves in and out of the tab strip normally.
- ARIA: `role="tablist"` on container; `role="tab"` on each button; `aria-selected` reflects active; associated panel has `role="tabpanel"` with matching `aria-labelledby`.
