**Public API:**

1. **`async snapshot(edges: IterableIterator<Edge>, snapshotTick: number): Promise<void>`** — Batched full rewrite.
   - Collect all edges into a flat params array: `[edgeKey, did_a, did_b, valence, weight, recency_tick, last_event_hash, snapshot_tick, ...]` per row.
   - Build a single `REPLACE INTO relationships (edge_key, did_a, did_b, valence, weight, recency_tick, last_event_hash, snapshot_tick) VALUES (?, ?, ?, ?, ?, ?, ?, ?), (?, ?, ?, ?, ?, ?, ?, ?), ...` with N value-tuple placeholders.
   - Compute `edge_key` as `` `${edge.did_a}|${edge.did_b}` `` (listener already sorted the DIDs).
   - Execute via `await this.pool.query(sql, params)`. Parameterize everything — D-9-05 gate 2 regex will accept `REPLACE INTO relationships` only inside this file.
   - If the edges iterator is empty, return early (do NOT send an empty VALUES clause — MySQL syntax error).
   - Wrap try/catch; on error log `{msg: 'relationships_snapshot_failed', tick: snapshotTick, err: err.message}` to the existing grid logger. DO NOT throw — fire-and-forget is a cache-write, losing one snapshot is recoverable via `rebuildFromChain`. Matches OQ-2 recommendation.

2. **`async loadSnapshot(): Promise<Edge[]>`** — Read-all for rebuild sanity and Inspector-startup fast-path.
   - `const [rows] = await this.pool.query&lt;RowDataPacket[]>('SELECT edge_key, did_a, did_b, valence, weight, recency_tick, last_event_hash FROM relationships');`
   - Coerce each row to `Edge`. `valence` and `weight` come back as strings from DECIMAL(4,3) in mysql2; convert with `parseFloat(row.valence)` — the 3-decimal round-trip is lossless because source values were already `.toFixed(3)` at canonical layer.
   - `recency_tick` comes back as `string | number` depending on mysql2 config; coerce to `Number(row.recency_tick)` (BIGINT UNSIGNED up to 2^53 is safe in Number).
   - Return the array. Empty relationships table returns `[]` (first-boot case).

3. **`scheduleSnapshot(edges: IterableIterator<Edge>, snapshotTick: number): void`** — Non-blocking wrapper used by Plan 04's launcher wiring:
   ```typescript
   setImmediate(() => {
       this.snapshot(edges, snapshotTick).catch((err) => {
           // Already caught inside snapshot(); this is defense-in-depth.
           // Do NOT rethrow — we are in a setImmediate callback.
       });
   });
   ```
   This is the ONLY place `setImmediate` appears in `grid/src/relationships/**`. It is allowed: `setImmediate` is not in the D-9-12 forbidden list (which covers `Date.now|performance.now|setInterval|setTimeout|Math.random`). Verify the Wave 0 determinism-source test still passes after adding this line.
