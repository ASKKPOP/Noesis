<threat_model>
This plan mitigates:
- **T-5-06 (HIGH)** — `trade.proposed` emitted AFTER reviewer. Mitigation: the rewritten handler emits `trade.proposed` BEFORE calling `reviewer.review()`, and `AuditChain.append()` is synchronous — the event is durably appended to the chain before the reviewer runs. Integration tests `trade-review-flow.test.ts` and `trade-review-abort.test.ts` assert `proposed[0].id < reviewed[0].id`.
- **T-5-02 (HIGH)** — Closed-enum bypass via stringly-typed payload at the JSON boundary. Mitigation: before calling `audit.append('trade.reviewed', ...)` on a fail verdict, the emit site asserts `VALID_REVIEW_FAILURE_CODES.has(verdict.failure_reason)` — throws if somehow a value escapes the TS union at runtime (defense in depth against future bugs in reviewer internals).
- **T-5-04 (MEDIUM)** — Reviewer payload leaks private data. Mitigation: the emit site uses 5 explicit keys `{ trade_id, reviewer_did, verdict, failed_check?, failure_reason? }` — no spread operator, no forbidden keywords. Plan 04 will add the D-12 regression test.
- **T-5-05 (MEDIUM)** — telosHash used as auth token. Mitigation: Phase 5 only shipped a structural-format check (Plan 01); no lookup or identity binding here. A code comment at the handler boundary re-confirms.

Partial mitigation from other plans:
- Plan 04 covers T-5-04 regression test, allowlist check for `trade.reviewed`, and T-5-07 zero-diff determinism.

```python
"""Deterministic Telos-hashing utility — Phase 5 REV-01 support (D-05).

Mirrors grid/src/audit/chain.ts:173-183 canonical-serialize-then-SHA-256 pattern.
Phase 7 TelosRegistry will import compute_active_telos_hash directly — keep the
canonical field order stable. ANY change to the canonicalization is a breaking
change for cross-phase hash compatibility.
"""

from __future__ import annotations

import hashlib
import json
from typing import Iterable

from .types import Goal


def compute_active_telos_hash(goals: Iterable[Goal]) -> str:
    """Compute deterministic 64-hex SHA-256 over the proposer's active goals only.

    Hashing input is canonical JSON (sort_keys=True, separators=(",", ":")) over a list
    of dicts containing ONLY the semantic goal fields. Inactive goals are excluded per
    PHILOSOPHY §5 (only the active-goal set constrains trades).

    Returns a lowercase 64-character hex digest (matches grid-side regex /^[a-f0-9]{64}$/).
    """
    active_goals = [
        {
            "description": g.description,
            "goal_type": g.goal_type.value,
            "status": g.status.value,
            "priority": g.priority,
            "progress": g.progress,
        }
        for g in goals
        if g.is_active()
    ]
    canonical = json.dumps(active_goals, sort_keys=True, separators=(",", ":"))
    return hashlib.sha256(canonical.encode("utf-8")).hexdigest()
```

        this.audit.append('trade.rejected', this.nousDid, {
            reason: result.error,
            nonce,
        });
        break;

    this.audit.append('trade.settled', this.nousDid, {
        counterparty,
        amount,
        nonce,
    });
