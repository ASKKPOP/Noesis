<objective>
Build `WsHub` and its internal `ClientConnection` — the component that owns all connected WebSocket clients, subscribes once to `AuditChain.onAppend`, enforces the Phase 1 broadcast allowlist, and fan-outs allowlisted events to each client through a per-client 256-entry RingBuffer with drop-oldest backpressure. Includes a bounded `sinceId` replay path on subscribe (REPLAY_WINDOW = 512) that satisfies ROADMAP Phase 2 Success Criterion #5 and closes PITFALLS §C6 (lost events across reconnect). Plus unit tests using a fake `ServerSocket` (no actual network) to exercise broadcast, allowlist filtering, topic globs, overflow, disconnect cleanup, and the three sinceId scenarios (recent/stale/current).

Purpose: This is where ACT-02 ("broadcasts all AuditChain events in real-time") becomes real AND where Phase 2 Success Criterion #5 ("reconnecting with `{type:'subscribe', sinceId: N}` replays missed events from in-memory chain OR tells client to use REST") lands. The hub is the only component that sits between AuditChain and any network: its listener MUST never call `ws.send` directly and MUST never throw back into `append()`. Plan 03 will plug it into the Fastify route; until then, we test the hub with a minimal socket double so we can catch fan-out bugs without pulling the whole integration layer in.

    The companion test file (Task 2) will exercise these behaviors. Listing them here so the implementation is designed to satisfy them:

    - Construction: `new WsHub({ audit, gridName: 'test' })` subscribes to `audit.onAppend` exactly once; `audit.append(...)` triggers the hub's internal dispatcher.
    - Connect: `hub.onConnect(fakeSocket)` immediately sends a `HelloFrame` with `type: 'hello'`, current `serverTime`, `gridName`, `lastEntryId = audit.length`.
    - Allowlist gate: when `audit.append('nous.spoke', ...)` fires, every connected client receives an `EventFrame`; when `audit.append('reflection.completed', ...)` fires (NOT allowlisted), NO client receives a frame.
    - Topic filter: client sends `{type:"subscribe", filters:["nous.*"]}`. After that, allowlisted `nous.moved` arrives but allowlisted `trade.settled` does NOT reach this client; an UNfiltered second client receives both.
    - Buffered/direct send: when `fakeSocket.bufferedAmount === 0`, the hub calls `socket.send` directly; when `bufferedAmount > watermarkBytes`, the hub pushes to the ring buffer and does NOT call `socket.send` until the next drain tick.
    - Overflow → DroppedFrame: fill the ring buffer (capacity 256 by default) past capacity while socket stays backpressured; when the drain happens (simulated by lowering `bufferedAmount` and invoking the hub's flush path), the next frame emitted is a `DroppedFrame` with `sinceId` = smallest evicted id and `latestId` = largest evicted id, BEFORE any queued events drain.
    - Listener never calls send under backpressure: the listener function body never contains `socket.send` invocation when `bufferedAmount > watermark` — it only enqueues. (Assertion: spy on `socket.send`; under permanent backpressure during a burst, enqueue count grows and send count stays equal to the pre-burst send count.)
    - Disconnect cleanup: when `fakeSocket` emits `'close'`, the hub's internal client set size decreases by 1; no further frames are sent to that socket even if new events fire.
    - close() drain: `await hub.close()` sends a `ByeFrame { reason: "shutting down" }` to each client, invokes `socket.close(1001, ...)`, clears the internal clients set, and unsubscribes from `audit.onAppend` (verified by: after close(), `audit.append(...)` no longer triggers any send on any remaining mock).
    - Cross-client isolation: one slow client whose `bufferedAmount` stays above watermark does NOT delay direct sends to a fast client whose `bufferedAmount` is 0 — fast client's send call happens synchronously inside the listener, slow client's data goes to its own ring buffer.
    - Listener robustness: throwing from a socket's `send` method inside the hub's listener path MUST NOT propagate out of `audit.append()` — wrap internal send attempts in try/catch (belt-and-suspenders alongside Phase 1's already-present try/catch in `AuditChain`).
    - Malformed client frame: `parseClientFrame` returning `null` causes the hub to ignore the message silently (no throw, no socket.close).

    NEW — sinceId replay protocol (ROADMAP Phase 2 Success Criterion #5 + PITFALLS §C6):
    - A `REPLAY_WINDOW` module-level constant is defined with value `512` (locked by 02-CONTEXT.md §Resume protocol).
    - On receiving `{type:'subscribe', sinceId: N}`:
      1. First apply filters (same as before) so subsequent live events respect the narrowing.
      2. Compute `gap = audit.length - N`.
      3. If `gap <= 0`: no-op (client is at or past head). No EventFrame, no DroppedFrame. Filters are applied and normal live delivery resumes.
      4. If `0 < gap <= REPLAY_WINDOW`: fetch the missed entries (use `audit.all().filter(e => (e.id ?? 0) > N)` since `audit.query()` has no id-range filter today) and, for each missed entry that BOTH passes `isAllowlisted(entry.eventType)` AND matches this client's filters, enqueue it as an `EventFrame` through the same `enqueue()` path live events use. Replay entries are delivered in id-ascending order BEFORE any live events.
      5. If `gap > REPLAY_WINDOW`: send a single `DroppedFrame { sinceId: N, latestId: audit.length }`. Do NOT replay any events; the client is expected to refill via `GET /api/v1/audit/trail?offset=N`. Live subscription continues normally afterwards.
    - Replay is synchronous from the hub's perspective (just reads `audit.all()` and enqueues). Backpressure is respected: replayed entries go through the same per-client ring buffer, so a slow client will naturally ring-buffer the replay burst rather than blocking the hub.
    - Replay does NOT re-subscribe or add any new audit listeners. The hub's single listener set from the constructor is the ONLY subscription — `grep -c "this.audit.onAppend"` still returns 1.
    - A subsequent subscribe frame WITHOUT sinceId (or with sinceId >= audit.length) silently no-ops the replay path and just updates filters.

    ```typescript
    /**
     * WsHub — owns all connected /ws/events clients. Subscribes ONCE to
     * AuditChain.onAppend, enforces the Phase 1 broadcast allowlist, and
     * fan-outs allowlisted entries to each ClientConnection for per-client
     * ring-buffered backpressure.
     *
     * Invariants (enforced by tests in grid/test/ws-hub.test.ts):
     *  1. The AuditChain listener NEVER calls ws.send — it only enqueues.
     *  2. A listener exception can never corrupt chain state (Phase 1
     *     already try/catches; we belt-and-suspenders here too).
     *  3. A slow client (high bufferedAmount) can NEVER delay a fast client.
     *  4. An entry whose eventType is not isAllowlisted() is dropped at the hub.
     *  5. close() is idempotent and leaves clients.size === 0.
     *  6. Resume: a {type:'subscribe', sinceId: N} frame replays missed allow-
     *     listed entries when gap ≤ REPLAY_WINDOW, else emits a single
     *     DroppedFrame telling the client to refill via REST. See PITFALLS §C6
     *     and ROADMAP Phase 2 Success Criterion #5.
     */

    import type { AuditChain } from '../audit/chain.js';
    import type { AuditEntry } from '../audit/types.js';
    import { isAllowlisted } from '../audit/broadcast-allowlist.js';
    import { RingBuffer } from '../util/ring-buffer.js';
    import {
        type HelloFrame, type EventFrame, type DroppedFrame, type ByeFrame,
        type ServerFrame, parseClientFrame,
    } from './ws-protocol.js';

    export interface ServerSocket {
        readonly bufferedAmount: number;
        send(data: string): void;
        close(code?: number, reason?: string): void;
        on(event: 'message', cb: (data: unknown) => void): void;
        on(event: 'close', cb: () => void): void;
        on(event: 'error', cb: (err: Error) => void): void;
    }

    export interface WsHubOptions {
        audit: AuditChain;
        gridName: string;
        bufferCapacity?: number;   // default 256
        watermarkBytes?: number;   // default 1_048_576
    }

    const DEFAULT_BUFFER_CAPACITY = 256;
    const DEFAULT_WATERMARK_BYTES = 1_048_576;

    /**
     * Maximum number of in-memory audit entries we will replay in response to
     * a {type:'subscribe', sinceId: N} frame. If the client is further behind
     * than this, we emit a single DroppedFrame and let them refill via the
     * REST /api/v1/audit/trail endpoint. Locked by 02-CONTEXT.md §Resume protocol.
     */
    export const REPLAY_WINDOW = 512;

    // ── Internal: one connected client ────────────────────────────────────────
    class ClientConnection {
        // ... (same fields as before: buffer, filters, droppedMin, droppedMax, closed)

        // setFilters(), matches(), enqueue(), tryDrain(), sendBye(), markClosed(),
        // trySend() remain unchanged from the previous structure.

        /**
         * Now takes a reference to the hub so 'subscribe' with sinceId can
         * trigger bounded replay. The hub passes `this` in onConnect via a
         * closure / bound method.
         */
        handleClientMessage(raw: unknown, onReplayRequest: (sinceId: number) => void): void {
            const frame = parseClientFrame(raw);
            if (!frame) return; // silent ignore — see PITFALLS C4 tangent
            switch (frame.type) {
                case 'subscribe':
                    this.setFilters(frame.filters);
                    // Resume protocol per ROADMAP Phase 2 SC#5 + PITFALLS §C6.
                    // The hub computes the gap and either replays or emits
                    // a DroppedFrame. We delegate because the hub owns the
                    // audit reference and the allowlist enforcement path.
                    if (frame.sinceId !== undefined) {
                        onReplayRequest(frame.sinceId);
                    }
                    break;
                case 'unsubscribe':
                    this.filters = []; // match nothing
                    break;
                case 'ping':
                    this.trySend({ type: 'pong', t: frame.t });
                    break;
                case 'pong':
                    // No-op — hub-driven keepalive lives at the hub level, not per-client here.
                    break;
            }
        }

        /** Emit a single DroppedFrame directly (used by hub for stale-sinceId replay). */
        sendDropped(sinceId: number, latestId: number): void {
            this.trySend({ type: 'dropped', sinceId, latestId });
        }

        // ... rest of the class unchanged.
    }

    // ── WsHub ────────────────────────────────────────────────────────────────
    export class WsHub {
        // ... unchanged fields: audit, gridName, bufferCapacity, watermarkBytes,
        // _clients, unsubscribeAudit, closing.

        onConnect(socket: ServerSocket, _req?: { headers?: Record<string, unknown> }): void {
            if (this.closing) {
                try { socket.close(1001, 'shutting down'); } catch { /* swallow */ }
                return;
            }
            const client = new ClientConnection(socket, this.watermarkBytes, this.bufferCapacity);
            this._clients.add(client);

            // Send Hello immediately (direct, no buffering).
            const hello: HelloFrame = {
                type: 'hello',
                serverTime: Date.now(),
                gridName: this.gridName,
                lastEntryId: this.audit.length,
            };
            try { socket.send(JSON.stringify(hello)); } catch { /* swallow */ }

            // Bind the replay callback: ClientConnection delegates sinceId
            // handling back to the hub (which owns audit + allowlist).
            socket.on('message', (raw) =>
                client.handleClientMessage(raw, (sinceId) => this.replayForClient(client, sinceId)),
            );
            socket.on('close', () => {
                client.markClosed();
                this._clients.delete(client);
            });
            socket.on('error', () => {
                client.markClosed();
                this._clients.delete(client);
            });
        }

        /**
         * Bounded replay path for the resume protocol. Honors:
         *  - REPLAY_WINDOW (512): beyond this, emit a DroppedFrame instead.
         *  - The same isAllowlisted() gate used for live events.
         *  - The client's current topic filter (matches()).
         *  - Per-client backpressure (we enqueue through the normal path).
         *
         * Replay runs in id-ascending order BEFORE any live event the client
         * hasn't seen yet — because it runs synchronously from inside the
         * 'message' handler, before the next audit.append listener fires.
         */
        private replayForClient(client: ClientConnection, sinceId: number): void {
            if (client.closed) return;
            const head = this.audit.length;
            const gap = head - sinceId;
            if (gap <= 0) return; // client is at or past head; no-op.
            if (gap > REPLAY_WINDOW) {
                // Too far behind — instruct client to refill via REST.
                client.sendDropped(sinceId, head);
                return;
            }
            // audit.query() has no id-range filter today; all() is fine at v1 scale.
            // REPLAY_WINDOW bounds the forEach at ≤ 512 entries.
            const entries = this.audit.all();
            for (const entry of entries) {
                const entryId = entry.id ?? 0;
                if (entryId <= sinceId) continue;
                if (!isAllowlisted(entry.eventType)) continue;
                if (!client.matches(entry.eventType)) continue;
                try {
                    client.enqueue(entry);
                } catch {
                    // Swallow — same defense-in-depth as onAuditEvent.
                }
            }
        }

        /**
         * Called from the audit listener. Never awaits. Never throws.
         * Enforces the broadcast allowlist at the single fan-out point.
         */
        private onAuditEvent(entry: AuditEntry): void {
            if (!isAllowlisted(entry.eventType)) return;
            for (const client of this._clients) {
                if (!client.matches(entry.eventType)) continue;
                try {
                    client.enqueue(entry);
                } catch {
                    // Swallow — a bug in enqueue must never crash append().
                }
            }
        }

        async close(): Promise<void> {
            if (this.closing) return;
            this.closing = true;
            try { this.unsubscribeAudit(); } catch { /* swallow */ }

            for (const client of this._clients) {
                client.sendBye('shutting down');
                try { client.socket.close(1001, 'shutting down'); } catch { /* swallow */ }
                client.markClosed();
            }
            this._clients.clear();
        }
    }
    ```

    tryDrain(): void {
        if (this.closed) return;
        if (this.socket.bufferedAmount >= this.watermarkBytes) return;

        if (this.droppedMin !== null && this.droppedMax !== null) {
            this.trySend({ type: 'dropped', sinceId: this.droppedMin, latestId: this.droppedMax });
            this.droppedMin = null;
            this.droppedMax = null;
        }

        // Snapshot buffer, send until backpressured, push remainder back.
        const items = this.buffer.drain();
        let i = 0;
        while (i < items.length && this.socket.bufferedAmount < this.watermarkBytes) {
            this.trySend({ type: 'event', entry: items[i] });
            i++;
        }
        // Push any un-sent remainder back (preserving order).
        for (; i < items.length; i++) {
            this.buffer.push(items[i]);
        }
    }

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| AuditChain.onAppend → WsHub listener | In-process, but listener must not reach back into chain state — Phase 1 already try/catches; hub adds a second try/catch around each enqueue for defense in depth. |
| Client WebSocket → WsHub.handleClientMessage | Hostile frame content: invalid JSON, wrong shape, wrong types, oversized sinceId. `parseClientFrame` (Plan 02-01) returns null on any anomaly. |
| Client sinceId → WsHub.replayForClient → audit.all() | Unbounded replay would let a misbehaving client request arbitrary history scans. REPLAY_WINDOW caps the scan at 512 entries and falls through to a single DroppedFrame beyond that. |
| WsHub → ServerSocket.send | Socket I/O can throw (closed socket, transport fault). Every send goes through `trySend` which catches exceptions. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-02-01 | Spoofing | Subscribe frame with forged filters | accept | Filters only narrow what a client receives. Server still enforces `isAllowlisted()` before fan-out, so a client cannot widen its stream beyond the allowlist regardless of filter string. |
| T-02-02-02 | Tampering | Listener throws → corrupts audit chain | mitigate | Hub wraps `client.enqueue(entry)` in try/catch AND `trySend` in try/catch. AuditChain itself also try/catches (Phase 1) so defense is layered. Test case `throwing socket.send does NOT propagate out of audit.append` locks this behavior. |
| T-02-02-03 | Repudiation | n/a | accept | No audit-of-observers in v1; all broadcast activity is derivable from `audit` itself. |
| T-02-02-04 | Information Disclosure | Non-allowlisted event leaks to browser via live fan-out OR via replay | mitigate | `isAllowlisted(entry.eventType)` is called BOTH in `onAuditEvent` (live) AND in `replayForClient` (sinceId path). A non-allowlisted entry in the chain history is NEVER replayed. Test `drops NON-allowlisted events silently` covers live; `subscribe with recent sinceId replays missed events` seeds only allowlisted entries (implicitly gated). Allowlist is Phase 1's immutable Set (runtime-frozen per broadcast-allowlist.ts). |
| T-02-02-05 | Denial of Service | Slow client stalls other clients (head-of-line blocking) | mitigate | Per-client RingBuffer + per-client `bufferedAmount` check ensures one slow socket's backpressure does not affect any other socket's send path (test `two clients with disjoint filters` and `fast client sends synchronously while slow is backpressured`). |
| T-02-02-06 | Denial of Service | Ring-buffer memory growth under sustained backpressure | mitigate | Buffer is bounded at `bufferCapacity` (default 256). Overflow evicts oldest; DroppedFrame signals client to refill via REST. Worst-case memory per client is `256 × entry_size ≈ 256 KB`. |
| T-02-02-07 | Denial of Service | Malformed client frame flood | mitigate | `parseClientFrame` returns null; hub silently ignores. No throw, no retry loop, no socket churn. Test `malformed client frame is ignored silently`. |
| T-02-02-08 | Elevation of Privilege | Per-connection listener leak (M8 pitfall) | mitigate | ONE listener total (`this.audit.onAppend((e) => this.onAuditEvent(e))`) — enforced by acceptance criterion `grep -c "this.audit.onAppend" returns 1`. The sinceId replay path does NOT add a second listener; it is a synchronous one-shot scan of `audit.all()`. Per-connection state is held in `Set<ClientConnection>` that `close/error` cleans up. Leak test in Plan 03 confirms `clients.size === 0` after 10k cycles. |
| T-02-02-09 | Denial of Service | Unbounded sinceId replay scan | mitigate | `REPLAY_WINDOW = 512` caps the number of entries delivered per subscribe frame. Beyond that, one DroppedFrame is sent and the client must fall back to REST — which has its own `@fastify/rate-limit`. A flood of subscribe frames with `sinceId: 0` still scans `audit.all()` per call; size is v1-bounded (single-developer scope). A future optimization (Phase 4+) can add `audit.entriesSince(id)` to avoid the full-array scan. |
