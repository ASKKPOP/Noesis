<objective>
Build the three stores that the UI (Plans 05–06) subscribes to via React's `useSyncExternalStore`. The stores are the single source of truth for:

1. **Firehose** — the rolling last 500 audit entries (ring-buffered in memory, DOM-capped later in Plan 05 to 100 rendered rows per UI-SPEC).
2. **Presence** — which Nous is in which region (driven by `nous.spawned` and `nous.moved` events).
3. **Heartbeat** — the tick count, last event arrival time, and a derived `live`/`stale` status (driven primarily by `tick` events but updated by ANY event so "last event N seconds ago" is accurate).

Each store exposes a `useSyncExternalStore`-compatible API: `subscribe(listener: () => void): () => void` + `getSnapshot(): TSnapshot`. Snapshots are referentially stable when nothing changed (no spurious re-renders).

Purpose: React components must NOT own business logic or event ingest. The stores are pure TypeScript classes testable without jsdom at all (though tests use Vitest with node env for speed).

Output:
- Three classes + one categorization helper, ~400 LOC total.
- Three test files covering ingestion, dedupe, idempotency, filter views, staleness detection.
- Zero React imports in store files — stores are framework-agnostic.
</objective>

    ```typescript
    import { categorizeEventType, type EventCategory } from './event-type';

    // Re-declared locally to avoid cross-plan import in Wave 2. Shape MUST stay
    // structurally compatible with dashboard/src/lib/protocol/audit-types.ts.
    export interface AuditEntry {
        id?: number;
        eventType: string;
        actorDid: string;
        targetDid?: string;
        payload: Record<string, unknown>;
        prevHash: string;
        eventHash: string;
        createdAt: number;
    }

    export interface FirehoseSnapshot {
        entries: readonly AuditEntry[];
        filteredEntries: readonly AuditEntry[];
        filter: ReadonlySet<EventCategory> | null;
        size: number;
    }

    export class FirehoseStore {
        private static readonly CAPACITY = 500;
        private readonly entries: AuditEntry[] = [];
        private readonly seenIds = new Set<number>();
        private filter: Set<EventCategory> | null = null;
        private listeners = new Set<() => void>();
        private cachedSnapshot: FirehoseSnapshot | null = null;
        private syntheticIdCounter = -1;

        ingest(incoming: AuditEntry[]): void;
        setFilter(filter: Set<EventCategory> | null): void;
        clear(): void;

        subscribe(listener: () => void): () => void;
        getSnapshot(): FirehoseSnapshot;  // MUST be referentially stable when nothing changed

        private invalidate(): void;  // clears cachedSnapshot, notifies listeners
    }

    export type FirehoseView = FirehoseSnapshot;
    ```

    Implementation notes:
    - `ingest`: for each entry, derive effective id (entry.id ?? this.syntheticIdCounter--), skip if already in seenIds, else push to entries + seenIds. If entries.length > CAPACITY, shift() oldest AND remove its id from seenIds. Call invalidate().
    - `getSnapshot`: if cachedSnapshot null, build {entries: this.entries.slice(), filteredEntries: computeFiltered(), filter: this.filter ? new Set(this.filter) : null, size: this.entries.length}. Freeze arrays with Object.freeze.
    - `setFilter`: store.filter = filter ? new Set(filter) : null; invalidate().
    - `computeFiltered`: if filter===null → return entries copy; if filter.size===0 → return []; else entries.filter(e => filter.has(categorizeEventType(e.eventType))).
    - `invalidate`: cachedSnapshot = null; for (const l of listeners) l().

    ```typescript
        private listeners = new Set<() => void>();
        private cachedSnapshot: PresenceSnapshot | null = null;
    ```
