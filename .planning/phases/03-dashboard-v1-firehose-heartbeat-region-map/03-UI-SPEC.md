# Phase 3 — UI Design Contract

> Visual and interaction contract for the first user-visible dashboard: firehose + heartbeat + region map at `/grid` on port 3001. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

**Design posture:** data-dense developer monitoring surface (think Grafana / Datadog / Honeycomb — not marketing). Dark theme default for long viewing sessions. Neutral and technical per FEATURES.md anti-feature list ("Gamified UI" explicitly banned — "closer to Honeycomb than to Stardew Valley"). Brand is "Noēsis" (Greek: *understanding*) — restrained, classical, no emoji, no rounded mascots.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (Tailwind 4 CSS-first via `@theme`) |
| Preset | not applicable |
| Component library | none (hand-rolled — 4 components total per 03-RESEARCH.md) |
| Icon library | `lucide-react` (^0.454.0) — used sparingly, monochrome only |
| Font (UI) | Inter (system fallback: `-apple-system, BlinkMacSystemFont, Segoe UI, sans-serif`) |
| Font (data / event IDs / timestamps / DIDs) | JetBrains Mono (fallback: `ui-monospace, SFMono-Regular, Menlo, monospace`) |

**Rationale for `tool: none`:** per 03-RESEARCH.md §"Standard Stack", Phase 3 ships 4 client components total. shadcn initialization cost (components.json, CLI registry, preset drift) exceeds value at this scale. Tailwind 4's `@theme` directive provides all tokenization without the registry layer. Re-evaluate in Phase 4 if inspector panel introduces form/dialog/dropdown primitives.

**Brand note:** no logo imagery in Phase 3 — page header is text only: `Noēsis` in Inter 22/600 followed by `/ grid` in JetBrains Mono 14/400 (dimmed). Greek etymology is the only brand signal needed for a dev tool.

## Spacing Scale

Declared values (all multiples of 4 — 8-point scale with one documented exception):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Intra-row padding inside a firehose event (icon → type pill → actor gap) |
| sm | 8px | Compact element spacing: chip internals, heartbeat digit gaps |
| md | 16px | Default element spacing: panel content padding, filter chip row |
| lg | 24px | Section padding: panel outer padding, map viewport margin |
| xl | 32px | Layout gaps: three-panel grid gutters on `/grid` |
| 2xl | 48px | Reserved for page-level margins around the `/grid` shell |

**Exceptions:**
- **Firehose row height: 28px** (not a multiple of 8). Justification: at 500 DOM-capped rows with `font-size: 12px` mono + `line-height: 1.3`, a 32px row wastes 4px per row × 500 = 2000 vertical pixels of scrollback. 28px is computed from `12px font + 8px vertical padding + 1px top border + tight line-height`. This is an ops-dashboard convention (Grafana Loki: 24px; Datadog Live Tail: 26px; Kibana Discover: 28px). Documented here so checker Dimension 5 does not flag it.

**Grid layout for `/grid` route:**

```
Desktop ≥1280px:
┌─────────────────────────────────────────────────────────────────┐
│  Header (Noēsis / grid  ·  Heartbeat widget right-aligned)       │  56px
├──────────────────────────────┬──────────────────────────────────┤
│                              │                                  │
│   Region Map  (SVG)          │   Firehose                       │
│   (left panel ~40% width)    │   (right panel ~60% width)       │
│                              │                                  │
│                              │   [filter chips row]             │
│                              │   ┌──────────────────────────┐   │
│                              │   │ scrolling event list     │   │
│                              │   │ (virtualized visually,   │   │
│                              │   │  DOM-capped at 100)      │   │
│                              │   └──────────────────────────┘   │
└──────────────────────────────┴──────────────────────────────────┘

Gap between panels: xl (32px). Panel inner padding: lg (24px).
Mobile/tablet <1280px: panels stack vertically, map collapses to 320px height.
```

## Typography

Four sizes, two weights. No italics. No underlines (except focus rings and hover links).

| Role | Size | Weight | Line Height | Font | Usage |
|------|------|--------|-------------|------|-------|
| Micro | 12px | 400 | 1.3 | JetBrains Mono | Firehose rows (timestamp, event type, actor DID, payload preview) |
| Body | 14px | 400 | 1.5 | Inter | Panel labels, filter chip text, tooltips, empty/error copy |
| Label | 14px | 600 | 1.4 | Inter | Panel titles ("Firehose", "Region Map"), filter section header |
| Heading | 16px | 600 | 1.4 | Inter | Region node labels on the map, heartbeat secondary text |
| Display | 22px | 600 | 1.2 | Inter | Page header "Noēsis", heartbeat tick counter ("t=1,847") |

**Heartbeat tick counter is the one tabular-numeric callout.** Render with `font-variant-numeric: tabular-nums` so the digit `1` doesn't collapse the counter width on each increment. Mono-family fallback acceptable.

**Rationale for monospace on firehose only:** Event IDs, DIDs (`did:noesis:sophia`), and ISO timestamps are fundamentally code-shaped data. Inter for chrome creates an explicit visual separation between "UI we read" and "data we scan" — the core Honeycomb/Grafana convention.

## Color

Base palette is dark. All values use OKLCH where Tailwind 4 defaults support it; hex provided for checker validation.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#0A0A0B` | Page background, map canvas background |
| Secondary (30%) | `#17181C` | Panel surfaces (firehose, map container, heartbeat widget), filter chip inactive background |
| Tertiary (~8%) | `#23252B` | Row hover, active filter chip, region node fill, inline code blocks |
| Accent (10%) | `#7DD3FC` (sky-300) | **Reserved — see explicit list below** |
| Destructive / stale | `#F87171` (red-400) | Heartbeat staleness indicator when `lastEventAgoMs > 2 × tickRate`, dropped-frame banner |
| Warning | `#FCD34D` (amber-300) | Reconnect-in-progress banner, backpressure `dropped` annotation |
| Success / live | `#4ADE80` (green-400) | Heartbeat dot when healthy, `grid.started` event pill |

### Accent reserved for:
1. Active filter chip text and 1px underline (not the whole background)
2. Live Nous marker stroke on the region map (the circle border for a Nous currently in a region)
3. The tick digit in the heartbeat (`t=<accent>1847</accent>`)
4. Hover border on a firehose row (1px accent border, not fill)
5. Focus ring on interactive elements (keyboard-accessible focus = 2px accent outline + 2px offset)

**Accent is NEVER used for:**
- Panel backgrounds or large fills
- Primary text
- Full-row highlighting in firehose (use tertiary `#23252B` for row hover fill)
- Filter chip backgrounds (use tertiary when active)

### Event-type color coding (firehose + map)

Each allowlisted event type (per `grid/src/audit/broadcast-allowlist.ts`) gets a single 8px colored dot on the left of its firehose row. This is the load-bearing scan signal — developers read the dot, not the word.

| Event class | Types (from allowlist) | Dot color | Rationale |
|-------------|------------------------|-----------|-----------|
| movement | `nous.spawned`, `nous.moved` | `#60A5FA` (blue-400) | Spatial events — cool blue matches "motion" convention |
| message | `nous.spoke`, `nous.direct_message` | `#C084FC` (violet-400) | Speech / social — purple convention from MMO ops consoles |
| trade | `trade.proposed`, `trade.settled` | `#FBBF24` (amber-400) | Economic activity — amber = coin/gold convention without being gamified |
| law | `law.triggered` | `#F472B6` (pink-400) | Rare, decisive events — warm color breaks firehose grey |
| lifecycle | `grid.started`, `grid.stopped`, `tick` | `#A3A3A3` (neutral-400) | Infrastructure — deliberately muted, not emergent |

These five classes map 1:1 to the filter chips specified in AUDIT-03 (`trade / message / movement / law`) plus `lifecycle` as an always-on infrastructure band that the user can toggle off in a second-row toggle.

**Contrast verification required in checker:**
- Body text (`#E5E7EB` at 14px/400) on dominant `#0A0A0B` = ~16:1 (AAA)
- Micro text (`#A3A3A3` at 12px) on secondary `#17181C` = ~7:1 (AAA for 14px, AA for 12px — acceptable)
- Accent `#7DD3FC` on dominant `#0A0A0B` = ~12:1 (AAA for any use)
- Destructive `#F87171` on secondary `#17181C` = ~5.5:1 (AA for body, PASS for indicator)

## Copywriting Contract

All copy is terse, technical, and assumes the reader is the developer who built the system (per FEATURES.md: "v1 audience is one developer who built it. Don't pave UX for users who don't exist yet"). No exclamation marks. No emoji in UI strings. No friendly tone — Honeycomb voice, not Slack voice.

| Element | Copy |
|---------|------|
| Page title (browser tab) | `Noēsis / grid` |
| Page header | `Noēsis` / `grid` (two-part, the slash is literal separator) |
| Heartbeat label (live) | `tick t=<N>  ·  last event <M>s ago` |
| Heartbeat label (stale) | `tick t=<N>  ·  stale <M>s  (no events for 2× tick rate)` |
| Heartbeat label (disconnected) | `disconnected  ·  reconnecting in <M>s` |
| Firehose panel title | `Firehose` (no subtitle) |
| Firehose filter row label | `Filter:` followed by chip buttons |
| Filter chips | `trade` · `message` · `movement` · `law` · `lifecycle` (all lowercase, matches event-type prefix) |
| Firehose empty state (no events yet) | Heading: `Waiting for events` / Body: `Grid is connected but no allowlisted events have arrived yet. Tick events appear every second.` |
| Firehose empty state (filter active, no matches) | Heading: `No events match` / Body: `No <filter> events in the last 500. Clear the filter or widen it.` |
| Firehose connecting state | Heading: `Connecting to Grid` / Body: `ws://<origin>/ws/events — attempt <N>` |
| Firehose disconnected (after bye frame) | Heading: `Grid shut down` / Body: `Received bye frame. Start the Grid and refresh.` |
| Firehose dropped-frame annotation | Inline row: `— dropped <K> events, refilling from /api/v1/audit/trail —` (rendered as a warning-colored full-width row, not a pill) |
| Region map panel title | `Region Map` |
| Region map empty state | Heading: `No regions loaded` / Body: `GET /api/v1/grid/regions returned empty. Is the Grid running?` |
| Region map error state | Heading: `Could not load regions` / Body: `<HTTP status>. Check Grid is on ws://<origin> and regions endpoint is registered.` |
| Region node label | Region name in Label style (14/600 Inter), followed by `<count> Nous` micro label below if count > 0 |
| Nous marker tooltip (hover) | `<nous.name> · <did>` (name in body, did in micro mono below) |
| Primary CTA | **Not applicable — Phase 3 is observer-only per REQUIREMENTS.md.** There are no forms, no submit buttons, no mutations. The only interactive controls are filter chips (toggle) and implicit page reload. |
| Destructive actions | **None in Phase 3.** Observer-only dashboard. No delete, no pause, no whisper. Phase 4 may introduce; Phase 3 has zero confirmation copy because nothing is destructive. |
| Generic error fallback (Next.js error boundary) | Heading: `Dashboard crashed` / Body: `Refresh to retry. If this persists, check browser console and Grid logs.` |

**Copy voice guidelines for any future strings in this phase:**
- Use lowercase `grid`, `nous`, `tick`, `did` as terms of art (matching the codebase).
- Refer to the server as `Grid` when speaking about the process and `grid` when referring to the `/grid` route or the conceptual substrate.
- Never say "simulation" in user-facing copy — `PHILOSOPHY.md` is clear that the Grid does not hide that it is artificial, but the dashboard also should not remind the user of gamification framing. Say "Grid," "tick," "Nous."
- Time units: `ms` for <1000ms, `s` for seconds. Never "seconds" spelled out in inline copy.
- Counts are always tabular-numeric. Never "a few" / "some" / "many".

## Interaction Contract

Only four interactions exist in Phase 3. Documenting them explicitly so the planner does not invent more.

| Interaction | Trigger | Response | Timing |
|-------------|---------|----------|--------|
| Toggle filter chip | Click / keyboard Space | Chip toggles active state; firehose re-renders filtered view from ring buffer | <16ms — pure client-side filter on in-memory array |
| Hover firehose row | Pointer over row | 1px accent border on row; row does not expand | CSS `:hover`, instant |
| Hover Nous marker on map | Pointer over `<g class="nous-marker">` | Tooltip appears with name + DID in upper-right of marker | 100ms delay, fade in 150ms |
| Nous moves on map | `nous.moved` event arrives | Marker `<g>` `transform: translate(x,y)` updates inside `flushSync`; CSS transition animates between positions | **CSS `transition: transform 150ms ease-out` on `<g.nous-marker>` — see 03-RESEARCH.md §"one-render-cycle movement"** |

**No click handlers in Phase 3.** Clicking a Nous marker or firehose row does nothing in v1. Phase 4 adds click-to-open-inspector; specifying non-behavior here prevents the executor from guessing.

**Keyboard accessibility baseline (required for checker):**
- Filter chips are `<button>` elements, tab-ordered left to right.
- Focus ring: 2px `#7DD3FC` outline + 2px offset on any focused element.
- Escape closes any open tooltip (no modals in Phase 3, but escape is a free safety net).
- `prefers-reduced-motion: reduce` disables the 150ms marker transition — marker jumps to new position instantly.

## Component Inventory

Four client components total (all under `dashboard/src/components/`):

| Component | File | Responsibility |
|-----------|------|---------------|
| `<Firehose>` | `Firehose.tsx` | Subscribes to `useFirehose()`, renders DOM-capped list of 100 most recent events matching active filter. Each row is a `<FirehoseRow>` subcomponent (lightweight, memoized by entry id). |
| `<Heartbeat>` | `Heartbeat.tsx` | Subscribes to `useHeartbeat()`. Renders tick counter (display), last-event-ago timer (label), live/stale/disconnected pill (accent/destructive color). Uses `setInterval(500ms)` to update the "ago" count between events. |
| `<RegionMap>` | `RegionMap.tsx` | Loads `/api/v1/grid/regions` once on mount. Renders SVG viewBox. Draws regions as `<g><circle/><text/></g>`. Subscribes to `usePresence()` and `useNousMotion()` to render Nous markers. |
| `<EventTypeFilter>` | `EventTypeFilter.tsx` | Five filter chips; controlled by a URL-state hook so filter survives reload. Chips render as `<button role="checkbox">`. |
