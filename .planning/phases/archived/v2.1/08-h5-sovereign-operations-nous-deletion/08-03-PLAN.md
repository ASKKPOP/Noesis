<objective>
Close AGENCY-05 on the dashboard side. Wire the H5 Sovereign delete affordance from the
Inspector, via a two-stage elevation flow (ElevationDialog → IrreversibilityDialog), to
the Grid's POST /api/operator/nous/:did/delete route created in 08-02. Add the
IrreversibilityDialog primitive with paste-suppressed DID-typed confirmation, verbatim
copy, and native <dialog>+showModal() semantics. Extend FetchError taxonomy to surface
410 Gone as 'nous_deleted'. Apply destructive visual treatment to firehose rows bearing
operator.nous_deleted events.

Purpose: Deliver the operator-facing UX for sovereign deletion. The backend (08-01, 08-02)
makes the event possible; this plan makes it actionable. UX discipline enforces the
philosophical claim — "There is no undo" — via paste suppression, exact-match typed
confirmation, verbatim warning copy, safer-default autoFocus on Keep this Nous, and
two-stage elevation that cannot be cancelled in one click.

Output: Working dashboard delete flow. Clicking an H5 delete button in the Inspector on
an active Nous opens ElevationDialog(H5); confirming opens IrreversibilityDialog;
pasting into the DID input is blocked; typing the DID exactly enables Delete forever;
confirming sends POST /delete; success toasts, refreshes state, auto-downgrades tier;
any error returns clear inline feedback. Deleted Nouses render State B (tombstoned
caption) and cannot be deleted twice. Firehose surfaces operator.nous_deleted with
destructive styling.
</objective>

    ```typescript
    export async function deleteNous(
        did: string,
        baseUrl: string,
        signal?: AbortSignal,
    ): Promise<OperatorResult<{ tombstoned_at_tick: number; pre_deletion_state_hash: string }>> {
        let resp: Response;
        try {
            resp = await fetch(`${baseUrl}/api/operator/nous/${did}/delete`, {
                method: 'POST',
                signal,
            });
        } catch (err) {
            if ((err as { name?: string })?.name === 'AbortError') throw err;
            return { ok: false, error: { kind: 'network' } };
        }
        if (resp.ok) {
            const data = await resp.json();
            return { ok: true, data };
        }
        return { ok: false, error: { kind: STATUS_TO_KIND_DELETE[resp.status] ?? 'network' } };
    }
    ```

    ```typescript
        onConfirm: () => void;

        openerRef?: React.RefObject<HTMLElement | null>;

    }: IrreversibilityDialogProps) {
        const dialogRef = useRef<HTMLDialogElement>(null);
        const [typed, setTyped] = useState('');
    ```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser ↔ Dashboard React process | User interaction (clicks, keyboard, paste). Untrusted: all input until validated. |
| Dashboard ↔ Grid REST API | POST /api/operator/nous/:did/delete + GET /api/introspect/:did. Cross-origin boundary with operator auth cookie. |
| Dashboard ↔ browser native `<dialog>` | Backdrop / ESC / close event semantics differ across browsers — relying on event timing requires discipline. |
| React component ↔ React component | IrreversibilityDialog trusts caller to manage `open` state correctly; caller trusts IrreversibilityDialog to fire onCancel on every close path. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-40 | S (Spoofing) | IrreversibilityDialog DID input | mitigate | Input value is confirmation-only; the targetDid prop is the source of truth. A spoofed input value simply fails exact-match comparison. No DID is taken from user input as the delete target. |
| T-08-41 | S (Spoofing) | openerRef prop | accept | Caller supplies the ref; focus restoration to a caller-controlled ref is expected behavior. If caller passes a malicious ref, the threat is already inside the React tree. |
| T-08-42 | T (Tampering) | Paste-bypass attempts via devtools | mitigate | `onPaste={e.preventDefault()}` blocks UI-level paste. Devtools can bypass via direct assignment to input.value — but this is a physical-access threat, not a UX-discipline threat. Accept the trade-off (PHILOSOPHY §operator agency: the console is not an adversarial environment). |
| T-08-43 | T (Tampering) | User races dialog close vs re-open with different DID | mitigate | `capturedDidRef.current = targetDid` snapshot at open (Phase 6 D-07 pattern). The exact-match check uses the captured value. Test asserts this. |
| T-08-44 | R (Repudiation) | User claims "I didn't confirm the delete" | mitigate | Audit chain entry operator.nous_deleted includes operator_id (authenticated DID) + target_did + pre_deletion_state_hash + tick. Audit is produced server-side (Grid, not dashboard) — client cannot repudiate what the backend recorded. |
| T-08-45 | I (Information disclosure) | deleteNous error response leaks implementation details | mitigate | OperatorFetchError is a discriminated union with only `{kind}` exposed. | T-08-46 | I (Information disclosure) | pre_deletion_state_hash appears in dashboard DOM | accept | Hash is on the successful response and currently goes only to the toast + audit firehose. The hash is public by design (D-03 — it's a Merkle-style commitment). Document in STATE.md. |
