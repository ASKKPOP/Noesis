# Phase 6 — UI Design Contract

> Visual and interaction contract for the first operator-agency surface: the persistent `<AgencyIndicator />` pill, the per-action `<ElevationDialog />` modal, and the H5 disabled-affordance placeholder in the Inspector. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

**Design posture:** Phase 6 extends the Phase 3 operator dashboard (`/grid`). Everything inherits the established "data-dense developer monitoring surface" voice (Honeycomb / Grafana / Datadog — not marketing). No gamification. No emoji. No rounded mascots. The Agency Indicator is load-bearing UX — it is the only always-visible reminder that the operator is *in* the system, not *above* it — but it must not shout. **Observer-but-not-controller is the core philosophical stance** (PHILOSOPHY §7); the chip's restraint embodies that.

**New in Phase 6 vs. v2.0 dashboard:**
1. `<AgencyIndicator />` — persistent pill, top-right of root layout, every route.
2. `<ElevationDialog />` — native `<dialog>` modal intercepting H2/H3/H4 actions before HTTP dispatch.
3. H5 affordance — visible-but-disabled button inside the Inspector drawer with tooltip.
4. Tier-colored `<Chip />` variant — extends existing primitive (`dashboard/src/components/primitives/chip.tsx`) with a `color` prop.

## Spacing Scale

Inherits the Phase 3 scale without change. Multiples of 4.

| Token | Value | Usage in Phase 6 |
|-------|-------|---------|
| xs | 4px | Chip internal gap (tier label ↔ tier code), dialog body ↔ action row |
| sm | 8px | Chip padding-y; dialog button row gap |
| md | 16px | Dialog body padding; tooltip internal padding |
| lg | 24px | Dialog outer padding; dialog min-width inset from viewport on mobile |
| xl | 32px | Header top-right offset for Agency Indicator (aligns with existing h1 baseline) |
| 2xl | 48px | Reserved — not used in Phase 6 |

**Exceptions:**
- **Firehose row height 28px** — inherited from Phase 3; still not a multiple of 8. Phase 6 does not touch firehose rows.
- **Elevation dialog min-width: 384px** (24rem) — not in the scale. Justification: dialog body carries two lines of 14px Inter body copy plus a 2-button row; 384px is the minimum that avoids awkward wrapping of the tier-name line ("Entering H3 — Co-decision. This will be logged."). Documented here so checker Dimension 5 does not flag.

**Agency Indicator placement (desktop ≥1024px):**
```
┌─────────────────────────────────────────────────────────────────┐
│  Noēsis  / grid              ...               [H1 Observer]    │
│  (h1 22/600)                                    (chip 12/600)   │
├─────────────────────────────────────────────────────────────────┤
```
- Pill sits at the right edge of the existing `<header>` (`grid-client.tsx` line 179), same vertical baseline as `h1`.
- Inline gap between the last header item and the pill: `xl` (32px).
- Pill right-edge flush with the page's right content margin (`p-4` from the `<main>` — 16px from viewport).

**Mobile <1024px:**
- Pill remains top-right, but header collapses to `flex-col` ordering on very narrow viewports (<640px); the pill stays visible as a second row, right-aligned. The indicator must never be hidden — SC#1 (every route renders `<AgencyIndicator />`).

## Color

Dark theme base (Phase 3). 60/30/10 distribution preserved at the page level.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#0A0A0B` | Page background, dialog backdrop at 80% opacity |
| Secondary (30%) | `#17181C` | Dialog surface, tooltip surface, chip inactive background (H1) |
| Tertiary (~8%) | `#23252B` | Chip hover state, dialog button hover fill |
| Accent (10%) | `#7DD3FC` (sky-300) | Focus rings only — see accent reserved list below |
| Destructive | `#F87171` (red-400) | H4 Driver tier chip color, dialog confirm button at H4 |
| Warning | `#FCD34D` (amber-300) | H3 Partner tier chip color, dialog confirm button at H3 |
| Neutral-active | `#60A5FA` (blue-400) | H2 Reviewer tier chip color, dialog confirm button at H2 |
| Muted (disabled) | `#525252` (neutral-600) | H5 Sovereign tier chip color (disabled), H5 Inspector button |

### Accent reserved for (Phase 6 additions):
1. **Focus ring** on chip, dialog buttons, tooltip-toggle keyboard focus — 2px outline + 2px offset (consistent with Phase 3 rule).
2. **Dialog close X affordance** hover state (if keyboard-accessible close icon is added per Claude's discretion).

**Accent is NEVER used for:**
- Chip background fill (see tier-to-color map below — accent is not a tier).
- Dialog confirm button primary fill (the fill color **is the tier color** — this is the load-bearing visual signal).
- Tier name text (tier name renders in neutral `#E5E7EB` or `#A3A3A3` — the color is carried by the chip outline/fill, not the text).

### Tier-to-color map (the load-bearing visual contract)

| Tier | Chip fill | Chip border | Chip text | Confirm button fill (dialog) | Contrast (chip text on fill) |
|------|-----------|-------------|-----------|------------------------------|-------|
| **H1 Observer** | `#17181C` (secondary) | `#2A2D34` (1px, neutral-800) | `#E5E7EB` (neutral-200) | N/A (no dialog at H1) | 12:1 AAA |
| **H2 Reviewer** | `#17181C` + 2px border `#60A5FA` (blue-400) | `#60A5FA` | `#E5E7EB` | `#60A5FA` with `#0A0A0B` text | 8:1 AAA |
| **H3 Partner** | `#17181C` + 2px border `#FCD34D` (amber-300) | `#FCD34D` | `#E5E7EB` | `#FCD34D` with `#0A0A0B` text | 13:1 AAA |
| **H4 Driver** | `#17181C` + 2px border `#F87171` (red-400) | `#F87171` | `#F87171` (text + border same color for urgency) | `#F87171` with `#0A0A0B` text | 6:1 AA (chip), 7:1 AAA (button) |
| **H5 Sovereign** | `#17181C` + 1px dashed border `#525252` | `#525252` | `#A3A3A3` with 1px `line-through` decoration | N/A (Phase 8) | 4.5:1 AA-minimum |

**Rationale for colors per tier (D-03 decision, locked):**
- **H1 neutral** — default, read-only, no action; blends with surface.
- **H2 blue** — calm, informational; read-only but logged. Does not scream.
- **H3 amber** — caution, reversible mutation (pause / law change). Matches existing `--color-warning` already used for reconnect banners (Phase 3 precedent); reader already associates amber with "pay attention, this is a soft boundary."
- **H4 red** — danger, irreversible mutation of sovereign state (Telos force). Matches existing `--color-destructive`; semantic weight mirrors Phase 3's staleness and dropped-frame banner.
- **H5 gray + strikethrough** — disabled, Phase 8 deferred; the strikethrough makes "not available" legible at a glance (per D-03 "strikethrough affordance").

**Color encodes the tier independently of the text** — a keyboard user who hears the tier name via `aria-label` gets the same signal as a sighted user who reads the color. Colorblind accessibility: the dialog title text (`Entering H3 — Co-decision`) and the `aria-label` carry the tier name in plain English, so no user depends on color alone.

### Contrast verification targets (checker must confirm):
- H4 chip border `#F87171` on `#17181C` = 5.5:1 (AA PASS for 14px body; PASS for 1px border convention)
- H3 chip border `#FCD34D` on `#17181C` = 11:1 (AAA)
- H2 chip border `#60A5FA` on `#17181C` = 6.3:1 (AA)
- H5 dashed border `#525252` on `#17181C` = 2.8:1 (fails AA for text, acceptable for **disabled decorative border** per WCAG 1.4.3 exemption — disabled controls are not required to meet contrast; tooltip + label redundantly carry meaning)
- Dialog confirm button text `#0A0A0B` on amber `#FCD34D` = 15:1 (AAA)
- Dialog confirm button text `#0A0A0B` on red `#F87171` = 8.5:1 (AAA)

### Empty / error / edge states

Phase 6 introduces no new empty states (the Agency Indicator always shows a tier, never empty; the elevation dialog is transient, never persistent). Error states inherit from calling components.

| Element | Copy |
|---------|------|
| Primary CTA (phase-level) | `Confirm` (elevation dialog) — the load-bearing phase CTA |
| Empty state heading | **Not applicable** — Agency Indicator always shows a tier; dialog is transient |
| Empty state body | **Not applicable** |
| Error state (H2/H4 Brain unreachable, per D-contextual) | Heading: `Action failed` / Body: `Brain is unreachable. Try again, or check Brain logs. No audit event was emitted.` — rendered by the calling component (Inspector for H4, memory query panel for H2), not the dialog |
| Destructive confirmation | `Entering H4 — Driver. This will be logged.` / Confirm button fill: `#F87171` |
| Other destructive actions | None in Phase 6. H5 Nous deletion is Phase 8 (AGENCY-05) and has its own irreversibility dialog with DID-typed confirm (out of scope here). |

**Destructive taxonomy for Phase 6:**
- **H2 (Reviewer):** read-only but audit-logged. *Not destructive; elevation still required because the action leaves a permanent trace.* Confirm button uses H2 blue, not red.
- **H3 (Partner):** reversible mutations (pause/resume, law add/amend/repeal). *Cautionary but not destructive.* Confirm button uses H3 amber.
- **H4 (Driver):** force-mutating a Nous's Telos is destructive to the Nous's sovereignty — this is the one tier in Phase 6 where the red-danger color is load-bearing. Confirm button uses H4 red.
- **H5 (Sovereign):** deleting a Nous — deferred to Phase 8 with its own separate dialog (AGENCY-05 DID-typed confirm). **Phase 6 ships only the disabled affordance**, not the actual confirmation flow.

### Copy voice guidelines (Phase 6 additions to Phase 3 list)

- Never say "mode" — tiers are tiers, not modes. ("mode-switch dialog" in AGENCY-04 is a REQ term describing the dialog's function, but UI copy must say "elevation" or "tier" — consistent with chip label and tooltip.)
- Never say "permission" — tiers are operator declarations, not system-granted permissions. Phase 6 is single-operator v2.1; the tier is self-declared by the operator at confirm time (D-14 explicitly: Grid trusts the stamp).
- The word "logged" (from AGENCY-04) is load-bearing — it is the honest admission that the operator is leaving a trace. Never soften to "recorded" or "tracked."
- Tier numbers (`H1`…`H5`) are always uppercase H + single digit, no hyphen, no space. The `H` is mono, the digit is mono.
- Tier names are always title-case single words: `Observer`, `Reviewer`, `Partner`, `Driver`, `Sovereign`.

## Interaction Contract

Four new interactions in Phase 6. Documenting them explicitly so the planner does not invent more.

| Interaction | Trigger | Response | Timing |
|-------------|---------|----------|--------|
| Open Agency tooltip | Hover chip (desktop) or click/focus chip (touch/keyboard) | Tooltip renders below-left of chip with the 5-row Agency Scale; active tier row bolded + accent left-border | 100ms hover delay, fade 150ms (matches Phase 3 Nous marker tooltip) |
| Close Agency tooltip | Escape key, click outside, pointer leaves both chip and tooltip | Tooltip fades out | 150ms |
| Trigger elevation | Click any H2/H3/H4 action control (inspector memory-query button, inspector force-telos button, law-change form submit, pause/resume controls) | Elevation dialog opens; target action's payload is held in a closure; focus trap activates on dialog; first focusable = Cancel (safer default) | Dialog appears within 16ms of click (no network I/O gate) |
| Confirm elevation | Click Confirm button or press Enter while Confirm has focus | Tier captured in closure; payload serialized with `{tier: "H{N}", operator_id, ...actionPayload}`; dispatch fires; dialog closes; tier auto-downgrades to H1 in state store | Close + downgrade fire **synchronously with dispatch call**, not after response (SC#4 / D-07 race-safety) |
| Cancel elevation | Click Cancel, press Escape, or click outside dialog | Dialog closes; **no dispatch**; tier stays at H1; payload discarded | <16ms |
| H5 disabled button interaction | Hover or keyboard-focus H5 Inspector button | Tooltip `Requires Phase 8` appears; button remains `disabled` and cannot receive click/Enter activation | Tooltip fades at 100ms |

**Elevation-race invariant (SC#4 / D-07) — visually:**

```
t=0   [click H4 action]
      ↓ open dialog, tier state UI still shows H1
      ↓ (any pointer motion during this window: tier state stays H1)
t=n   [click Confirm]
      ↓ tier="H4" captured in closure
      ↓ body = {tier: "H4", operator_id, target_did, ...} serialized
      ↓ fetch() fires with body already built
      ↓ setAgencyTier('H1') fires in the same tick (finally block or post-dispatch)
t=n+1 [UI shows H1 again]
      (response arrives later; committed audit row has tier="H4" per D-07)
```

The `finally` block auto-downgrade is what makes the indicator chip appear to "flash" the elevated tier for <1 frame during a fast-dispatch action. **That flash is intentional** — it gives the operator visual confirmation the tier really was used — but it does not compromise race-safety because the tier is already serialized into the request body before the downgrade fires (D-07 implementation note).

**Keyboard accessibility baseline (required for checker):**
- Chip is a `<button type="button">` element, tab-ordered after the page h1 and any preceding header controls.
- Dialog is a native `<dialog>` element with `showModal()` — inert background, focus trap, Escape = cancel (native behavior, no polyfill needed).
- First focusable in dialog = Cancel button (not Confirm — the safer default for accidental Enter-key trigger).
- H5 disabled button is keyboard-focusable (`tabindex="0"` on the button, with `aria-disabled="true"` and `disabled`) so screen readers announce it as "disabled — requires Phase 8."
- `prefers-reduced-motion: reduce` — tooltip fade and dialog open animations collapse to instant display.
