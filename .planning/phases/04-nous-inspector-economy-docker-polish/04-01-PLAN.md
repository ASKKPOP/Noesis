    ```typescript
    /**
     * Atomically transfer Ousia between two Nous.
     * Returns a tagged result; performs no mutation unless success is true.
     */
    transferOusia(
        fromDid: string,
        toDid: string,
        amount: number,
    ):
        | { success: true; fromBalance: number; toBalance: number }
        | { success: false; error: 'not_found' | 'insufficient' | 'self_transfer' | 'invalid_amount' } {
        if (!Number.isInteger(amount) || amount <= 0) {
            return { success: false, error: 'invalid_amount' };
        }
        if (fromDid === toDid) {
            return { success: false, error: 'self_transfer' };
        }
        const from = this.records.get(fromDid);
        const to = this.records.get(toDid);
        if (!from || !to) {
            return { success: false, error: 'not_found' };
        }
        if (from.ousia < amount) {
            return { success: false, error: 'insufficient' };
        }
        from.ousia -= amount;
        to.ousia += amount;
        return { success: true, fromBalance: from.ousia, toBalance: to.ousia };
    }
    ```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| brain → grid (BrainAction.trade_request) | Brain-authored metadata arrives via JSON-RPC; may be malformed or hostile |
| nous-runner → audit.append | Any unchecked field ends up on the WS broadcast; privacy leak risk |
| brain → nous-runner.transferOusia | Arbitrary counterparty DIDs can be supplied; must validate before mutating state |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Tampering | `trade.settled` payload shape | mitigate | Exactly three keys (`counterparty`, `amount`, `nonce`) enforced by code; integration test grep-asserts `Object.keys(payload).sort()` matches the frozen set (D8, Pitfall 4) |
| T-04-02 | Information Disclosure | Free-form `text`/`name` leaking into audit payload | mitigate | Trade handler never reads `action.text` and never writes text/name to audit; broadcast allowlist (Phase 1) is the second line of defense but this is the first |
| T-04-03 | Denial of Service | Malformed BrainAction crashing append | mitigate | Per-branch try-free validation returns `trade.rejected` instead of throwing; listener try/catch isolation from Phase 1 backs this up |
| T-04-04 | Elevation of Privilege | Nous transferring from someone else's account | mitigate | `transferOusia` is called only with `this.nousDid` as `from`; sender cannot override |
| T-04-05 | Repudiation | Lost trade settlement in audit | mitigate | `trade.settled` is appended inside `executeActions` synchronously; audit is the chained, hashed source of truth |
| T-04-06 | Spoofing | Unknown `counterparty` DID | mitigate | `transferOusia` returns `not_found` → emits `trade.rejected` rather than creating balance out of thin air |
</threat_model>
