# Phase 4 — UI Design Contract

> Visual and interaction contract extending Phase 3 with the Nous Inspector drawer, the Economy tab, and a tab bar on `/grid`. Generated by gsd-ui-researcher in `--auto` mode — every answer is sourced from CONTEXT.md (16 locked decisions), RESEARCH.md, or inherited from 03-UI-SPEC.md. No new libraries, no new fonts, no new animations, no shadcn.

**Design posture (inherited from Phase 3):** dark-theme developer monitoring surface. Grafana / Honeycomb idiom. Dense, neutral, technical. Monospace for data, Inter for chrome. No gamification. No emoji. No exclamation marks. Observer-first (Inspector is READ-ONLY per REQUIREMENTS "Out of Scope").

**Phase 4 surfaces added:**
1. **Tab bar** on `/grid` — two tabs, querystring-driven (`?tab=firehose` default, `?tab=economy`).
2. **Inspector drawer** — right-side sheet (~420px), URL-hash-persisted (`#nous=did:noesis:<name>`), opened from firehose row or map marker click.
3. **Economy panel** — second tab: Balances table, Recent Trades table, Shops list.

### Accent reserved for (Phase 4 — additive to Phase 3 list):

Phase 3's five accent uses are inherited unchanged. Phase 4 adds:

6. **Drawer header underline** — 1px accent border under the Nous name row, nowhere else in the drawer chrome.
7. **Focus rings inside the drawer** — same 2px outline + 2px offset as Phase 3. Applies to every focusable element in the drawer (close button, tab-able links in memory summaries, counterparty DID links in trade rows).
8. **Active tab indicator** — 2px accent bottom-border on the active tab. Inactive tabs have no border. (Tab text itself stays `#E5E7EB` regardless of active state — the 2px border is the sole signal.)
9. **Selected-Nous marker outline on the region map** — when `SelectionStore.selectedDid` matches a marker's DID, that marker gets an additional 2px accent outer-stroke in addition to its existing live-marker stroke. Only one marker at a time carries this.

**Accent is NEVER used for (Phase 4 additions):**
- Balance column text, trade amount text, or any monetary numeric (kept at body default `#E5E7EB` — Ousia is not a brand-accent moment).
- Drawer section titles (use Label role at default color).
- Tab inactive backgrounds (use secondary).

**Inspector memory-kind badge colors.** Memory `kind` values (e.g. `observation`, `dialogue`, `goal_resolved`) are NOT event-classes, but rendering them requires a scannable visual. **Decision:** all memory-kind badges use the `lifecycle` neutral `#A3A3A3` with the badge text showing the raw kind string. Rationale: we do not invent a second color palette for inner-life taxonomy; memories are observations, not allowlist events. If the kind string happens to overlap with an allowlist class (e.g. `trade`), the planner may choose to re-use that class color — but only when the string is a verbatim match to an allowlist prefix. Default is lifecycle-neutral.

### Trade.settled flash (subtle, not animation-heavy)

When a `trade.settled` WS frame arrives:
- The **proposer's balance row** in the Balances table briefly shows a 1px left-border in `--color-destructive` (red) for 600ms then fades out.
- The **counterparty's balance row** gets a 1px left-border in `--color-success` (green) for 600ms then fades out.
- The Recent Trades table prepends the new row; the new row mounts with `opacity: 0.6` fading to `opacity: 1.0` over 200ms. No slide, no glow.
- `prefers-reduced-motion: reduce`: disable the opacity fade; the row simply appears. The 1px left-border still renders but stays for 600ms without fade.

**This is deliberately quieter than any "toast/notification" pattern.** Honeycomb voice: the data changed, you can see it changed, you do not need to be told it changed.

### Drawer focus-trap contract (WAI-ARIA, hand-rolled)

Per CONTEXT specifics and RESEARCH.md "Hand-rolled vs react-focus-lock":

- On drawer mount: `useEffect` records `document.activeElement` as `originEl`, moves focus to the close button.
- Tab / Shift+Tab inside the drawer cycles through focusable descendants in DOM order. The first and last focusable elements are linked — Tab from last wraps to first, Shift+Tab from first wraps to last.
- Escape key: calls `SelectionStore.clear()`. Handler attached to `document.addEventListener('keydown', ...)` inside the drawer's `useEffect`, removed on unmount.
- Backdrop click: same as Escape. Backdrop is a `<div role="presentation">` covering the full viewport behind the drawer.
- On unmount: focus restored via `originEl.focus()` if `originEl` is still in the DOM; else no-op.
- `aria-modal="true"` on the drawer `<aside>`; `role="dialog"` on the drawer; `aria-labelledby` points to the drawer header's Nous name.
- Rest of the page behind the backdrop gets `aria-hidden="true"` while the drawer is open.

### Tab bar roving tabindex

- The tab bar is a `<div role="tablist">` containing two `<button role="tab">` elements.
- Only the active tab has `tabindex="0"`; the inactive tab has `tabindex="-1"`.
- Arrow-left / arrow-right inside the tablist moves focus AND activates the sibling tab (activate-on-focus per the simpler WAI-ARIA pattern for 2-tab bars).
- Home / End go to first / last tab.
- The tab panel below has `role="tabpanel"` and `aria-labelledby` referencing the active tab's id.

### URL hash parser contract (security)

Per CONTEXT specifics: `#nous=<did>` MUST validate against `/^did:noesis:[a-z0-9_\-]+$/i` before entering the store. Malformed hash → stripped silently, no store write, no drawer open, no error copy shown. This prevents XSS via hash injection.

### URL querystring contract

- `?tab=economy` → Economy tab active.
- Missing param OR `?tab=firehose` OR any other value → Firehose tab active (default, not an error).
- Tab switching uses `router.replace(url, {scroll: false})` to avoid scroll reset and avoid adding a history entry per click (dev tool; users don't want back-button history bloat).

### prefers-reduced-motion

- Drawer open/close fade: disabled; drawer snaps in and out.
- Trade row prepend opacity fade: disabled; row appears instantly.
- Balance row delta flash (1px left-border): **still renders** — the border is informational, not motion. It holds for 600ms without fade.
- Phase 3 marker CSS transition: already respects reduced-motion per 03-UI-SPEC.

|------|------|---------|
| `useHashSync(key)` | `lib/hooks/use-hash-sync.ts` | Two-way binding between `window.location.hash` param `<key>=<value>` and a setter callback; sanitizes via regex allowlist |

### Data flow (inspector)

```
click row/marker
    └─> SelectionStore.setSelected(did)
            ├─> useHashSync writes `#nous=<did>` to URL
            └─> Inspector subscribes via useSelection()
                    ├─> mounts loading state
                    └─> fetch(`/api/v1/nous/${did}/state`)
                            ├─> 200 → render 4 sections
                            ├─> 503 → <EmptyState> "Brain unreachable"
                            └─> 404 → <EmptyState> "Nous not found"
```

### Data flow (economy)

```
tab switches to ?tab=economy
    └─> <EconomyPanel> mounts
            ├─> fetch(`/api/v1/grid/nous`) → BalancesTable
            ├─> fetch(`/api/v1/economy/trades?limit=20`) → TradesTable
            ├─> fetch(`/api/v1/economy/shops`) → ShopsList
            └─> subscribe to WS `trade.settled` frames
                    └─> on each frame:
                            ├─> re-fetch balances + trades
                            ├─> flash proposer row red 600ms
                            └─> flash counterparty row green 600ms
```
