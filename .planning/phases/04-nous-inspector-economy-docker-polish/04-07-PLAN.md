    ```dockerfile
    # syntax=docker/dockerfile:1.7

    # ── Stage 1: Builder ─────────────────────────────────────────────────────────
    FROM node:22-alpine AS builder

    WORKDIR /app

    # NEXT_PUBLIC_* must be available at BUILD time — Next inlines them into the
    # client bundle. Passing via `docker run -e` is too late. See D10 / Pitfall 1.
    ARG NEXT_PUBLIC_GRID_ORIGIN=http://localhost:8080
    ENV NEXT_PUBLIC_GRID_ORIGIN=${NEXT_PUBLIC_GRID_ORIGIN}

    # Copy workspace manifests first (layer caching)
    COPY package.json package-lock.json* ./
    COPY dashboard/package.json ./dashboard/

    RUN npm ci

    # Copy dashboard source
    COPY dashboard ./dashboard

    # Build in standalone mode (next.config.mjs → output: 'standalone')
    RUN npm run build --workspace=dashboard

    # ── Stage 2: Runtime ─────────────────────────────────────────────────────────
    FROM node:22-alpine AS runtime

    WORKDIR /app

    ENV NODE_ENV=production \
        PORT=3001 \
        HOSTNAME=0.0.0.0

    RUN addgroup --system --gid 1001 nodejs \
     && adduser  --system --uid 1001 nextjs

    COPY --from=builder --chown=nextjs:nodejs /app/dashboard/.next/standalone ./
    COPY --from=builder --chown=nextjs:nodejs /app/dashboard/.next/static    ./dashboard/.next/static
    COPY --from=builder --chown=nextjs:nodejs /app/dashboard/public          ./dashboard/public

    USER nextjs
    EXPOSE 3001

    # Standalone layout puts the server entrypoint at dashboard/server.js
    CMD ["node", "dashboard/server.js"]
    ```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| compose build → image contents | Build-arg leakage risk if secrets passed via ARG |
| runtime container → host | Must run as non-root |
| dashboard → grid over compose network | Internal hop; origin of trust is compose-managed network, not the internet |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-33 | Information Disclosure | Secrets baked into client bundle | mitigate | Only `NEXT_PUBLIC_*` vars are passed as ARGs; .env.example forbids pattern `SECRET_*` with an explicit comment |
| T-04-34 | Elevation of Privilege | Root process in runtime image | mitigate | Non-root `nextjs:nodejs` user; USER directive before CMD |
| T-04-35 | Tampering | Rebuild-less env change leaking stale URLs | mitigate | .env.example warns about NEXT_PUBLIC_* build-time semantics; compose passes it as BOTH arg + env for parity |
| T-04-36 | Denial of Service | Dashboard crash loop before grid is ready | mitigate | `depends_on.grid.condition: service_healthy` gates startup |
| T-04-37 | Spoofing | Cross-origin attack via CORS allowlist drift | mitigate | Dashboard publishes :3001 exactly — matches the frozen CORS allowlist from D9; no widening needed |
</threat_model>
