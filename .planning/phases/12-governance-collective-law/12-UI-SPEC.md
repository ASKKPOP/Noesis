# Phase 12 — UI Design Contract
## Governance & Collective Law Dashboard Page

> Visual and interaction contract for the Governance page at
> `dashboard/src/app/grid/governance/page.tsx`. This is a read-only operator
> view of the commit-reveal proposal lifecycle (VOTE-01..07). Operators see
> aggregate vote counts; H5 sees per-Nous voting history. No vote, propose,
> or tally affordance exists anywhere in the Dashboard — governance is
> intra-Nous only (VOTE-05).
>
> Generated by gsd-ui-researcher. Sources: 12-CONTEXT.md (D-12-01..D-12-11),
> 12-RESEARCH.md §Standard Stack, dashboard/tailwind.config.ts,
> dashboard/src/app/globals.css, Phase 9 (09-UI-SPEC.md) as the canonical
> pattern precedent for stand-alone `/grid/*` pages with SWR polling, and
> Phase 6 (06-UI-SPEC.md) for tier-gated content patterns.
> Mode: --auto (all decisions pre-locked in CONTEXT.md + RESEARCH.md +
> codebase scan; no user questions needed).

## Typography

Inherits the four-size scale established across Phases 6–11. No new sizes.

| Role | Size | Tailwind | Weight | Line-Height | Usage in Phase 12 |
|------|------|----------|--------|-------------|------------------|
| Section heading | 14px | `text-sm` | 600 | 1.25 | Page h1 "Governance"; proposal detail heading; H5 modal heading |
| Body / label | 12px | `text-xs` | 400 | 1.5 | Proposal metadata (status, tick range, counts); tally result labels; footnote disclaimers |
| Meta / caption | 11px | `text-[11px]` | 400 | 1.4 | Proposal ID (truncated); outcome caption; quorum note; H5 voting history timestamps |
| Monospace ID / hash | 12px | `font-mono text-xs` | 400 | 1.3 | `title_hash` display; `proposal_id` truncated; `commit_hash` in H5 history rows |

**Weight discipline:** Only 400 (regular) and 600 (semibold). No 500, no 700. Matches Phases 6–11.

**Monospace usage:** `proposal_id` UUIDs, `title_hash` hex, `commit_hash` hex, and DID strings render in JetBrains Mono. Numeric tally counts (yes/no/abstain integers) render in Inter (`text-xs`) to mark them as semantic content, not identifiers.

**Line-height discipline:** `leading-relaxed` reserved for multi-sentence warning copy only (inherited Phase 8). Proposal status copy and tally results use default `leading-normal` (1.5).

### Outcome status scale (NEW — Phase 12, scoped to governance surfaces)

Phase 12 binds three proposal outcome colors. These reuse existing Tailwind palette steps — no new `globals.css` tokens required.

| Outcome | Tailwind Token | Hex | Usage |
|---------|---------------|-----|-------|
| `open` / active | `text-sky-400` / `bg-sky-400/10` | `#38BDF8` | Proposal status badge when `status = 'open'` |
| `passed` | `text-green-400` / `bg-green-400/10` | `#4ADE80` | Proposal status badge + tally outcome label when `outcome = 'passed'` |
| `rejected` | `text-neutral-400` / `bg-neutral-700/20` | `#A3A3A3` | Proposal status badge when `outcome = 'rejected'` |
| `quorum_fail` | `text-amber-400` / `bg-amber-400/10` | `#FBBF24` | Proposal status badge when `outcome = 'quorum_fail'` |
| `tallied` (generic) | `text-neutral-400` / `bg-neutral-800` | `#A3A3A3` | Fallback for any tallied proposal before outcome known client-side |

**Existing event category color for governance events in firehose:**
The firehose already maps `law.triggered` to `bg-pink-400/10 text-pink-300` (`--color-event-law`). Phase 12's four new event types (`proposal.opened`, `ballot.committed`, `ballot.revealed`, `proposal.tallied`) belong to the same `law` category — they use the same pink badge without any new color token.

### Accent reserved for (explicit list)

Brand accent (`#7DD3FC`) is reserved for:
1. `/grid/laws` link text on passed proposals ("View enacted law →")
2. Focus ring on the H5 voting history button (consistent with Phase 6 H2 elevation pattern)
3. Any future tab underline if a "Governance" tab is added to the main nav

Outcome colors (`sky`, `green`, `amber`) are reserved for:
1. Proposal status badge only (the `<span>` pill on each proposal row)
2. Tally outcome label in the proposal detail view

Outcome colors MUST NOT appear on:
- Buttons (use existing tier-chip colors or neutral)
- Page headings
- Footnote copy
- H5 voting history rows (those use neutral palette only)

### Primary CTA label

Phase 12 is read-only. No primary action CTA exists. The sole interactive affordance is the H5 voting history button: **`View voting history (H5)`** — which opens a read-only modal (no mutation, no confirmation required).

### Destructive confirmations

**None.** Phase 12 is entirely read-only from the dashboard perspective. No delete, no propose, no vote, no tally trigger. VOTE-05 mandates zero write affordance for operators. No `IrreversibilityDialog` usage.

### Copywriting style guide (inherited from Phase 9)

- No exclamation marks. No emoji. No "Oops" or "Uh oh."
- Active voice: "When a Nous opens a proposal…" not "A proposal will be shown when opened…"
- Named nouns: "Nous" (capital N), "Grid" (capital G), "tick" (lowercase), "Governance" (capital G as page proper noun).
- Numeric precision: yes/no/abstain counts are integers — no decimal formatting.
- Avoid temporal vague words: "deadline tick {N}" not "deadline soon". Replay rigs operate on ticks, not wall-clock.

### Surface 1a: Proposals List

**Component (NEW, client):** `dashboard/src/app/grid/governance/governance-dashboard.tsx`

**SWR polling:** `useSWR('/api/v1/governance/proposals', { refreshInterval: 2000 })` — clones Phase 9 relationships `refreshInterval` pattern. `refreshInterval` applies only when `status === 'open'` proposals exist in the response; when all proposals are tallied, polling can slow to `10000` (10s). Implementation detail for the hook, not a hard spec constraint.

**Proposal row anatomy:**

```
┌──────────────────────────────────────────────────────────────┐
│ [open]  e3a1…f29c  Deadline: tick 500  Commits: 3  Reveals: 2│
└──────────────────────────────────────────────────────────────┘
```

- Status badge: `<span>` pill with outcome color (see §Color outcome scale)
- `proposal_id` truncated: first 4 chars + `…` + last 4 chars (e.g. `e3a1…f29c`) — `font-mono text-[11px] text-neutral-600`
- Deadline tick: `text-xs text-neutral-400`
- Commit/reveal counts: `text-xs text-neutral-400`
- Row is `cursor-pointer` with `hover:bg-neutral-800/50` — clicking selects the proposal for the detail pane

**Tailwind classes (proposal row):**


## Interaction Contracts (State Machines)

### Governance page state machine

```
┌────────────────────────────────────────────────────────┐
│ /grid/governance loaded                                  │
│ SWR key: '/api/v1/governance/proposals' refreshInterval 2s
└──────────────┬─────────────────────────────────────────┘
               │
               ├──▶ loading → render 3× skeleton rows (animate-pulse)
               │
               ├──▶ error → EmptyState + error copy from §Copywriting
               │
               └──▶ success:
                    │
                    ├─ data.proposals.length === 0 → EmptyState "No proposals yet."
                    │
                    └─ data.proposals.length > 0 → render proposal list rows
                         │
                         └──▶ click row → selectedId = proposal.proposal_id
                              │
                              └──▶ render detail pane:
                                   ├─ tier === 'H1': title_hash + metadata + tally (if tallied) — NO body text
                                   ├─ tier === 'H2': same + body text fetched via separate SWR key
                                   ├─ tier === 'H5': same as H2 + "View voting history (H5)" button
                                   └─ outcome === 'passed': "View enacted law →" link

Click "View voting history (H5)" (tier === 'H5' only)
  ↓ open VotingHistoryModal with selectedId
  ↓ fetch GET /api/v1/governance/proposals/:id/ballots/history {tier:'H5'}
  ↓ render ballot rows OR empty-state
  ↓ Close → modal unmounts → focus returns to opener button
```

### Keyboard navigation

| Key | Context | Action |
|-----|---------|--------|
| Tab | Page loaded | Cycle proposal rows → detail pane controls → H5 button |
| Enter / Space | Proposal row focused | Select proposal (sets selectedId) |
| Enter / Space | H5 history button focused | Open VotingHistoryModal |
| Escape | VotingHistoryModal open | Close modal, restore focus to opener button |
| Enter / Space | `View enacted law →` link | Navigate to `/grid/laws` |

## Accessibility Contract

| Requirement | Implementation | Standard |
|-------------|---------------|---------|
| Proposals list ARIA | `role="listbox"` on `<ul>`; `role="option"` + `aria-selected` on each row | WAI-ARIA Listbox |
| Outcome is never color-only | Badge pill text ("open"/"passed"/"rejected"/"quorum fail") always present alongside color | WCAG 1.4.1 |
| H5 modal `role="dialog"` | Plain dialog (not `alertdialog` — read is non-destructive) + `aria-labelledby` | WAI-ARIA 1.2 |
| H5 modal focus trap | Native `<dialog>` + `showModal()` — browser-native inert background | WCAG 2.4.3 |
| H5 modal focus restoration | openerRef pattern — capture on open, restore on close | WCAG 2.4.3 |
| No timeout on modal | Modal stays open indefinitely | WCAG 2.2.1 |
| Contrast — all text | All instructional copy uses `text-neutral-400` or higher on `bg-neutral-950` (≥7.2:1) | WCAG 1.4.3 (AA ≥4.5:1) |
| `aria-live` for loading | `<div aria-live="polite" className="sr-only">Loading proposals…</div>` | WCAG 4.1.3 |
| Proposal ID readability | Truncated UUID displayed as `code` element in monospace; full ID in `title` attribute | WCAG 1.3.1 |
| H5 voting history DID readability | Full DID displayed in `<code>` + `break-all` for long DID strings | WCAG 1.3.1 |

**`aria-modal`:** Native `<dialog>` with `showModal()` sets `aria-modal="true"` implicitly — do not add manually (inherited Phase 8 + Phase 9 discipline).

## Error States

### Governance page — proposals list

| Trigger | Copy | Recovery |
|---------|------|----------|
| 503 | `Grid unreachable. Governance data will reload when the connection recovers.` | SWR auto-retry (2s interval) |
| 500 | `Governance data could not be loaded. Reload to retry.` | Manual reload |
| No proposals | `No proposals yet.` / `When a Nous opens a proposal, it will appear here.` | Informational |

### Proposal detail pane

| Trigger | Copy | Recovery |
|---------|------|----------|
| 403 tier_insufficient (body) | `H2 required to view proposal body.` | Elevate tier via AgencyIndicator |
| 410 proposal_not_found | `This proposal no longer exists.` | Return to list (clear selectedId) |
| 503 body fetch | `Proposal body temporarily unavailable.` | SWR auto-retry |

### VotingHistoryModal (H5)

| Trigger | Copy | Recovery |
|---------|------|----------|
| 403 tier_insufficient | Modal does not open (guard on tier check before fetch) | N/A — UX-prevented |
| 503 | Inline: `Grid unreachable. Vote records temporarily unavailable.` (`text-xs text-neutral-400`) | Manual close; retry on reopen |
| 404 no ballots | Same as empty state: `No vote records for this proposal.` | Informational |
